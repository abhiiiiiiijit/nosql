MongoDB Cheat Sheet

Core Concepts

  • Document database (JSON-like)
  • Collections instead of tables
  • No schema enforcement
  • _id field as primary key
  • BSON data format

Key Advantages

  • Flexible schema
  • Horizontal scaling (Sharding)
  • High availability (Replica sets)
  • Rich query language
  • Aggregation framework

Common Use Cases

  • Real-time analytics
  • IoT applications
  • E-commerce platforms
  • Content management
  • Mobile apps

CRUD Operations

  • Create: insertOne(), insertMany()
  • Read: find(), findOne()
  • Update: updateOne(), updateMany()
  • Delete: deleteOne(), deleteMany()

Index Strategies

  • Single field indexes
  • Compound indexes
  • Multikey indexes
  • Text indexes
  • TTL indexes

Aggregation Pipeline

  • $match: Filter documents
  • $group: Group documents
  • $sort: Order results
  • $project: Reshape documents
  • $lookup: Join collections

Architecture Features

Replica Set

  • Primary + Secondaries
  • Automatic failover
  • Data redundancy

Sharding

  • Horizontal scaling
  • Shard key selection
  • Query router (mongos)
